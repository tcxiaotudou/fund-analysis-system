# 终端风格页面浅色工作台主题设计

## 背景

当前系统的终端风格页面统一使用深黑背景、青色高亮和深色 Ant Design 覆盖样式。用户反馈黑暗模式有点刺眼，并确认希望所有终端风格页面一起改为浅色工作台主题。

## 目标

将所有复用 `terminal-dashboard` / `TerminalPage` 的页面统一调整为低刺激的浅色工作台风格。页面应保留现有紧凑的信息密度、驾驶舱结构和交互行为，只降低暗色高对比和霓虹高亮带来的视觉疲劳。

## 范围

包含以下页面和区域：

- 首页终端驾驶舱：市场概览、决策摘要、信号矩阵、操作控制台等终端卡片。
- 非首页 `TerminalPage` 页面：动量策略、MA 策略、RSI 分析、RSI 回测、基金推荐、基金组合、ETF 管理、系统配置。
- 终端主题内的 Ant Design 覆盖：卡片、表格、分页、按钮、输入框、日期选择器、下拉菜单、筛选菜单、弹窗、消息提示、警告提示。
- 图表视觉：坐标轴、网格线、图例文字、Tooltip 外观。

不包含以下内容：

- 不改业务逻辑、数据请求、回测流程、路由和表格字段。
- 不增加主题切换功能。
- 不引入新的组件库或新依赖。
- 不重做页面布局或信息架构。

## 视觉方向

采用浅灰蓝背景和白色内容面板。强调色从高亮青色改为低饱和蓝绿，保留“终端/工作台”的精密感，但去掉黑底和强发光效果。

核心色彩意图：

- 页面背景：浅灰蓝，降低整页亮度反差。
- 内容面板：白色或近白色，使用细边框区分层级。
- 主文本：深灰蓝，保证表格和正文可读。
- 次级文本：中性灰蓝，避免过淡。
- 强调色：低饱和蓝绿，用于标题、主按钮、进度和选中状态。
- 买入/卖出/警告等语义色保持绿、红、橙，但改成适合浅色背景的低刺激版本。

## 实现设计

主要改动集中在 `frontend/src/assets/css/index.css`：

- 调整 `.app-layout-terminal`、头部、侧栏和菜单的背景与边框，使全局壳层进入浅色工作台风格。
- 重设 `.terminal-dashboard` 的 CSS 变量，包括背景、面板、边框、文本、弱文本、强调色和语义色。
- 调整 `.dashboard-command-bar`、`.terminal-panel`、`.terminal-page .ant-card` 等通用容器，改为浅色面板、细边框和轻阴影。
- 同步调整终端主题下的表格、分页、输入框、按钮、弹窗、下拉、日期选择器、消息提示和图表样式，避免出现浅色页面里混入深色控件。
- 保留现有类名和组件结构，使所有终端页面自然继承新主题。

如局部组件仍写死深色 `rgba(2, 6, 23, ...)` 或 `#0f172a`，只在视觉必要时替换为浅色主题变量或浅色等价色，不做无关重构。

## 数据与状态

本次调整不改变 React 状态、API 调用、URL 参数、表格排序筛选、回测弹窗提交逻辑或图表数据构造。所有交互行为应与改动前一致。

## 错误处理

不新增兜底逻辑、不吞掉错误、不模拟成功路径。若页面资源、接口或图表加载失败，应继续按现有方式暴露错误或空状态。

## 验证方式

完成实现后需要验证：

- 构建命令通过。
- 动量策略页面首屏渲染为浅色工作台主题，卡片、表格、图表、按钮、弹窗风格一致。
- 首页终端驾驶舱渲染为同一浅色主题，三栏布局和信息密度不被破坏。
- 至少检查一个移动宽度视口，确认内容不重叠、不溢出。
- 浏览器控制台无相关运行时错误。

## 风险

终端主题 CSS 覆盖面较广，可能存在少量组件使用硬编码深色。实现时需要用浏览器逐页抽查关键页面，发现明显割裂的深色控件时同步修正。
